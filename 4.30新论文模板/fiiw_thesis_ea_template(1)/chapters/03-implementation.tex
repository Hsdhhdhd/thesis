\section{Method for Objective 2: Labelled Sitting Dataset and Classifier Evaluation}\label{sec:labelled_dataset}

\subsection{On-Body Labelled Sitting Dataset}

The detection analysis uses XIAO data only and treats the target as the presence or absence of pelvic motion rather than anatomical pelvic-angle estimation. The setup prepared before each field session is shown in Figure~\ref{fig:c6_equipment_overview}: multiple wearable units, paired phones, and Shimmer reference sensors used for the validation subset described in Section~\ref{sec:shimmer_validation}.

Five volunteers wore the prototype near the sacrum during an extended urban route that included walking from Group T in Leuven, train travel, a meal period, the return journey, and the walk back. The study was approved by the KU Leuven PRET committee under \texttt{G-2025-10284-R2(MIN)} (project ``Daily Monitoring of Pelvic Rotation Using Wearable IMUs''), dated 5 January 2026; all participants gave informed consent. No demographic or clinical variables were analysed in this thesis, so the pilot results cannot be stratified by age, sex, body composition, sitting habits, or history of low back pain.

Participant identifiers P1--P5 in this chapter and in Chapter~\ref{ch:results} refer only to the five pilot-study participants. They should not be confused with protocol phases or previous pilot labels. The protocol was designed to collect labelled XIAO seated segments for the classifier dataset while also collecting paired Shimmer/XIAO field recordings from the same outing.

\subsection{Windowing and Feature Extraction}\label{sec:segmentation_features}

For the offline classifier, labelled seated segments are loaded from the Android CSV exports and converted into window-level feature vectors. During loading, non-numeric samples are dropped, rows are sorted by epoch timestamp, and exact duplicate rows are removed.

Within each clean segment, raw IMU data are divided into sliding windows of 2~s with a 1~s step, retaining only windows with at least 50 samples. Each retained window inherits the label of its source segment. The IMU is configured at 208~Hz, but \(I^2C\) overhead and RTOS scheduling latency in the polling firmware reduce the effective read rate in clean field segments; this is handled by requiring a minimum sample count rather than assuming a fixed number of samples in every window.

The three derived signals were acceleration magnitude \(\|\mathbf{a}\|\), gravity-removed acceleration magnitude \(|\|\mathbf{a}\|-9.80665|\), and gyroscope magnitude \(\|\boldsymbol{\omega}\|\). For each of the six raw channels and these three derived magnitude signals, ten time-domain statistics were computed: mean, standard deviation, minimum, maximum, range, root-mean-square, median, interquartile range, mean absolute first difference, and standard deviation of the first difference. This yields a 90-dimensional feature vector per window.

\subsection{Classifier Evaluation Protocol}\label{sec:classifier_protocol}

Five classical machine-learning classifiers were trained on the same 90-dimensional feature matrix from participants 1--4 and evaluated on participant 5: logistic regression (LR), support vector machine with RBF kernel (SVM), decision tree (DT), random forest (RF) with 500 trees, and KNN. The split is person-independent, so no windows from the held-out participant were used for training. For LR, SVM, and KNN, standardisation was included inside the scikit-learn pipeline and fitted only on participants 1--4 before being applied to participant 5. The reproducibility settings are listed in Table~\ref{tab:ch6_model_settings}. The KNN value \(k=7\) was selected from candidate values \(k\in\{3,5,7\}\) using leave-one-participant-out checks on participants 1--4 only.

\section{Method for Objective 3: Shimmer/XIAO Signal Comparability Evaluation}\label{sec:shimmer_validation}

\subsection{Co-Located Comparison Setup}

For the co-located comparison, the custom wearable and the Shimmer reference were fixed to the same adhesive mounting surface and moved as a single rigid body (Figure~\ref{fig:shimmer_mounting}). The Shimmer was configured at 100~Hz, and the XIAO IMU was configured at 208~Hz. The paired setup was manually moved through repeated rotations, swinging motions, and static periods. This recording checks whether the XIAO captures the same time-varying inertial pattern as the reference sensor under a shared movement input.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.54\linewidth]{shimmer_placement.png}
    \caption{Co-located mounting of the custom wearable and the Shimmer reference sensor. The two devices were attached to the same adhesive surface so that they moved together during the comparison.}
    \label{fig:shimmer_mounting}
\end{figure}

\subsection{Signal Alignment and Metrics}

The co-located signals were resampled to a common time base and aligned by cross-correlation. Pearson correlation at zero lag describes waveform similarity without extra temporal correction. Maximum cross-correlation describes the strongest similarity after temporal alignment. The estimated lag is reported in seconds to indicate whether one data stream is shifted relative to the other. Normalised root-mean-square error (NRMSE) is normalised by the standard deviation of the Shimmer signal and is used as an amplitude-disagreement measure. The aim is trend-level comparison for movement detection, not clinical equivalence or anatomical angle validation.

\subsection{Field-Worn Paired-Recording Subset}\label{sec:pilot_detection}

The field-worn Shimmer/XIAO comparison used the same pilot outing described in Section~\ref{sec:labelled_dataset}. Paired reference recordings were planned for all five participants, but failed or incomplete Shimmer logs were excluded from the field-worn Shimmer comparison. Clean labelled XIAO seated segments could still be retained for the classifier dataset even when the corresponding Shimmer reference log was unusable.

\subsection{Field-Worn Trend Protocol}

The field-worn trend comparison uses the two usable paired recordings from the pilot outing. In these recordings, XIAO samples were timestamped at the Android receiver, whereas Shimmer used its own logger clock. Epoch timestamps therefore defined the overlap but did not guarantee exact clock synchronisation. Field signals were resampled to 10~Hz, and gaps longer than 0.5~s were not interpolated. This avoids treating long XIAO recording gaps as smooth movement.

The field comparison uses two magnitude signals rather than axis-by-axis agreement: acceleration motion magnitude \(a_m(t)=\left|\lVert\mathbf{a}(t)\rVert-g\right|\) and gyroscope magnitude \(\omega_m(t)=\lVert\boldsymbol{\omega}(t)\rVert\). These signals are less sensitive to small orientation differences between sensors during body wear. The same correlation, maximum cross-correlation, lag, and NRMSE metrics used in the co-located comparison are reported for these long field trends.

\section{Method for Objective 4: Embedded Deployment Timing and Memory Evaluation}\label{sec:compact_timing_protocol}

\subsection{Compact KNN for Embedded Deployment}\label{sec:compact_knn}

A full KNN implementation is memory-heavy because it stores the training examples as its reference set. The deployed firmware therefore uses two reductions: a 12-feature subset (Table~\ref{tab:ch6_compact_knn_features}) and KMeans prototype compression. The feature subset is dominated by gyroscope and acceleration-range information, which matches the expected signal difference between static sitting and deliberate pelvic movement. The training and held-out evaluation procedure used to fix the final model configuration is described in Section~\ref{sec:classifier_protocol}.

Before compression and inference, each selected feature is standardised using mean and standard-deviation values exported from the training pipeline. Within each labelled class on the standardised training windows, KMeans (\texttt{random\_state=42}, \texttt{n\_init=10}) extracts 64 centres, yielding 128 stored prototypes in total. Inference compares the current window against these prototypes using a distance-weighted KNN with \(k=7\). The exported firmware header contains the selected feature order, scaling values, prototype coordinates, class labels, and KNN setting.

\subsection{Firmware Timing and Haptic Trigger Protocol}

The compact prototype-based KNN was evaluated on the same held-out participant and then implemented in firmware. The deployed firmware samples the IMU at a 5~ms polling interval, accumulates non-overlapping 2~s windows, computes the 12 compact features online, and classifies each window as static sitting or pelvic rotation. When pelvic rotation is detected, the inactivity timer is reset; otherwise, after 10~min without a detected pelvic-rotation window, the DRV2605L drives the vibration motor for one second.

Per-window inference latency was measured on the XIAO nRF54L15 Sense using the hardware cycle counter (\texttt{k\_cycle\_get\_64}) across 24 consecutive non-overlapping 2~s windows. The timing measurement includes both the compact feature extraction path and the KNN distance computation for the stored prototypes. Flash and RAM usage were taken from the firmware build output. These measurements evaluate compute feasibility only; power consumption and battery lifetime were not measured.
